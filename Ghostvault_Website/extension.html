GhostVault Browser Extension

A privacy‑first browser extension that protects your accounts and your data with encrypted password management, breach detection, tracker blocking, and privacy analysis.

Add to Chrome Get for Edge Back to overview

Key features

Secure Passwords

AES‑GCM encryption, PBKDF2 key derivation, master password never stored.

Breach Detection

Checks emails/passwords via HaveIBeenPwned (optional API key).

Tracker Blocking

Blocks common analytics/ads with MV3 declarative net request rules.

T&C & Cookies

Analyzes terms and cookie dialogs for privacy risks.

Phishing Checks

Flags suspicious sites and patterns.

Modern UI

Clean, accessible dark mode interface.

One‑click install

GhostVault will be listed on the Chrome Web Store and Edge Add‑ons. Use the buttons above for easy installation.

Developer Mode (optional)

  1. Open chrome://extensions (Chrome) or edge://extensions (Edge).
  2. Enable “Developer mode”.
  3. Click “Load unpacked” and select the GhostVault_Extension folder.
  4. Pin GhostVault and click the icon to open the popup.

Later, you can package for the Chrome Web Store. For now, dev mode is perfect for testing.

Quick Start

  • Open the popup and set your master password.
  • Visit a login page, sign in, and save credentials when prompted.
  • Use the Breach Checker to test an email with HIBP.
  • Toggle tracker blocking and other settings in the options page.

Using GhostVault

Saving passwords

When you submit a login form, GhostVault detects it and offers to save your credentials securely. Your data is encrypted with AES‑GCM; the master password is never stored.

Auto‑fill

On supported pages, previously saved logins can be auto‑filled from the popup.

Breach checks

Enable HaveIBeenPwned checks by adding your API key in settings, then check an email for known breaches directly from the popup.

T&C and cookies

On pages with privacy dialogs, GhostVault analyzes language and highlights potential privacy‑unfriendly patterns before you accept.

Security architecture

Encryption

AES‑GCM with 256‑bit keys, PBKDF2 (100k iterations), random salt & IV per encryption.

Local‑first

Data stored in IndexedDB; master password only in memory while needed.

Privacy

No tracking or analytics. External calls only for breach checks you initiate.

Transparency

Readable code, clear error handling, and explicit permissions.

Troubleshooting

Extension not loading

  • Developer mode is enabled.
  • All files are present in the folder.
  • Reload the extension and check the console for errors.

Login detection doesn’t trigger

  • Give the extension access to the current site.
  • Ensure the page uses standard HTML form elements.

Security tips

  • Use a strong, unique master password.
  • Regularly check for breaches.
  • Keep your browser up to date.
